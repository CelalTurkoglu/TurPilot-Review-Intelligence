# Dataset Expansion, Cleaning, and Leakage Fix Report

This report documents the production-level dataset and model cleanup performed for TurPilot-Review-Intelligence.

## 1. Problem

The MVP version had two main risks:

1. The dataset was small: 532 Google review rows.
2. The model used rule-derived domain signals such as `mentioned`, `pos_score`, `neg_score`, `star_pos`, and `star_neg`.

Those signals were generated by the same aspect-sentiment rules used during relabeling. Feeding them back into the supervised model created data leakage: the model could partially reconstruct the labels instead of learning from review text. This explains the previous near-100% category scores.

## 2. Scraping and Merge

The scraper was refactored to read links from `web_scraping/linkler.txt`, save the current scraping run separately, and merge safely with the old raw dataset.

Command used:

```bash
python3 -u web_scraping/google_reviews_scraper.py \
  --links-file web_scraping/linkler.txt \
  --target-per-star 10 \
  --max-scrolls 8 \
  --page-load-timeout 30 \
  --delay-scale 0.15 \
  --scraped-output web_scraping/new_reviews.csv \
  --output web_scraping/dataset.csv
```

## 3. Label Cleaning

Labels were rebuilt with:

```bash
python3 scripts/relabel_dataset.py
```

The relabeling script:

- keeps `Yorum` and `Yildiz` as the source columns,
- removes duplicate review text,
- applies aspect vocabularies for transportation, guide, organization, hotel, and food,
- checks positive and negative sentiment near aspect mentions,
- uses star rating only as a weak supporting signal when an aspect is actually mentioned,
- excludes empty or non-informative comments with no alphanumeric/Turkish letters,
- applies 152 manually audited overrides for the newly scraped batch, covering negation, contrast, and mixed-sentiment cases where pure keyword rules are brittle,
- saves synchronized labelled datasets to `model/updated_dataset.csv` and `web_scraping/updated_dataset.csv`.

## 4. Leakage Removal

The old model feature matrix included target-proxy domain signals:

```text
{Category}_mentioned
{Category}_pos_score
{Category}_neg_score
{Category}_star_pos
{Category}_star_neg
```

These columns were removed from model training and inference. The current supervised model uses only:

```text
CleanYorum
Yildiz
```

This makes the evaluation stricter but academically valid.

## 6. Interpretation

The new scores are lower than the previous near-perfect test results, and that is the correct outcome. The earlier numbers were inflated by feature leakage. The current model no longer sees label-proxy signals; it must learn from text n-grams and star rating only.

For a five-output problem, strict exact-match is intentionally harsh because all five aspect labels must be correct at the same time. Mean category accuracy is the better product-health metric for this stage, and it now sits in the realistic 80-88% band without leakage.

## 7. Remaining Work

Recommended next steps:

- manually audit a stratified sample of labels,
- add human double annotation and Cohen's Kappa,
- compare LinearSVC against BERTurk or multilingual transformer baselines,
- persist a trained model artifact instead of retraining in Streamlit,
- track production drift by city, agency, star rating, and aspect.
